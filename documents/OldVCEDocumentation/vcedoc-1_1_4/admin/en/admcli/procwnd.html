admcli: process window

Description of process window

This window visualizes target PROCESSES running on target hosts. State of each processes is monitored through VCE monitor API. You can sort entries by clicking headline of each column. Youc can choose multiple lines at the same time. It's convenient to execute the same commands against multiple processes in one action.

Description of shown entries

Alert
It turns to "!" when pre-set condition is satisfied.
Id
unique process ID number given by admsv. This is used when searching through log to determine process.
Addr
IP address of the host that has target process.
Name
Name of target process.
Tag
Tag string for your own use. You can set this entry by admsv configuration file or admcli.
State
state of the process. There are 3 states: DEAD,INIT,ALIVE
It'll be shown red if DEAD or INIT.
  • DEAD : process is down. (vce-monitor cannot connect to it.)
  • INIT : process is now in the way starting.
  • ALIVE : process is healthy.
    • Type
    Type of process. now it's always 1.
    Port
    Port number of the process' service.
    AdmPort
    Port number for Admin tool, not service
    Ping
    Round trip time of ping request from admsv to the process. This is highly influenced by process' activity, for example if the process is performing a big calcuration, ping value will bigger.
    Ping graph
    graph of Ping value. You can configure it by [Tool]-[Configuration]-[Graph]
    Login
    Number of concurrent user of the process. This is obtained by VCE monitor API. THe process have to return it by get_login monitoring command.
    Login graph
    Graph of Login number.
    Session
    Number of TCP sessions that are holded by target process.
    Session graph
    Graph of Session.

    Controlling process window

    You can manipulate process window by these ways:


    • On/Off Auto update of process window
    • Update process window once
    • Execute command against the process
    • Execute registered command
    • Start/Stop/Restart process
    • Edit tag string
    • Create/Delete process entry
    • Configure startup command

    On/Off Auto update of host window
    You can on/off auto update of host window.
    Update host-window once
    You can force admcli to update host window.
    Execute command against the process
    Execute Monitor command Result will be shown in result window
    Commands can be selected from pop-up menu by right click, or just you can type the command by yourself.
    Execute registered command
    not implemented yet.
    Start/Stop/Restart process
    You have to configure admsv configuration file to perform this, or use configure startup command beforehand.
    Edit tag string
    Edit tag string.
    Create/delete new process entry
    Create and delete entry.
    Configure startup command
    Configure starup command for starting each process that'll be monitored. admcli cannot start target process without configuring this correctly.
