Host window

Description of host window

This window visualizes current state of RSV running on target machine. You can sort entries by clicking headline of each column. You can select multiple lines at the same time, and it can be useful when you want to perform the same commands against multiple hosts at once.

Description of each items

Alert
It turns to "!" when pre-set condition is satisfied.
Id
ID number of each host that is given by admsv.
Addr
IP address of each host.
Name
Name of the host.
Tag
Tag string for your own use. You can set this entry by admsv configuration file or admcli.
State
state of the host. There can be 3 states : DEAD,INIT,ALIVE
It'll be shown red if DEAD or INIT.
This indicates some problem is happening on that host.
  • DEAD : RSV is not running on target host
  • INIT : RSV is on the way booting
  • ALIVE : RSV is healthy.
    • LoadAvr
    Load average of the host. if 1000, LoadAverage is 1.0.
    LoadAvr graph
    Graph of load average. You can customize through [Tool]-[Configuration]-[Graph]
    Ping
    Round-trip time between admsv and host.
    Ping graph
    Graph of Ping. You can customize this by [Tool]-[Configuration]-[Graph]

    Controlling host window

    You can manipulate host window in these ways:

    • On/Off Auto update of host window
    • Update host-window once
    • Execute command against the host
    • Edit tag of the host
    • Create a new host
    • Delete an existing host

    On/Off Auto update of host window
    You can on/off auto update of host window.
    Update host-window once
    You can force admcli to update host window.
    Execute command against the host
    Execute UNIX command on that host. RSV running on target machine will receive the input command, and exec(). Result will shown in result window.
    Edit tag of the host
    Eidt host TAG.
    Create a new host
    Add a new host to be monitored. You put new IP address, and Tag string.
    Delete a host
    Delete an existing host.
