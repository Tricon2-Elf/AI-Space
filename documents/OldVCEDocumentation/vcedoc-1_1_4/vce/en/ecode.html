VCE's Error Codes and Error Messages

There are two methods to identify an error in VCE; getting an error number by vce_get_last_error function and referring the output strings in the error log (vce_errout). These methods are explained in below.

VCE's Error Log Message

There are four kinds of message stored in a log by VCE: Fatal, Warning, Bug, and Note. All messages are numbered; Fatal error messages start from 1000, Warning message starts from 4000, Bug message starts from 7000, and Note message starts from 10000. Numbers would be increased in future version of VCE, but existing messages will not be deleted nor modified (detailed description may be changed). The following messages are for VCEgg 0.2. These are defined as macro by vce.h.
Because descriptions of these messages may be changed in the future, always use the error number such as "Fatal: 1101" rather than the English description as a clue to analyze a log with backup tools and administration tools.

Fatal Error Messages

Fatal error messages are generally caused with the wrong usage of VCE by application or with insufficient configuration of the operating system (problem with port, free memory, permission problem, etc.) An error can be solved by carefully tracking down the cause.

  • "Fatal:1000 cannot initialize winsock\n"

    Cannot initialize Winsock.

  • "Fatal:1003 array table scarcity (init_array)\n"

    The number of remaining tables of array is insufficient. Too many vce_init_array. Check if array leak is occurred.

  • "Fatal:1010 MALLOC fail (index=%d)\n"

    MALLOC (wrapper of malloc) failed. Mostly due to insufficient memory.

  • "Fatal:1011 MALLOC fail (index=%d)\n"

    MALLOC (wrapper of malloc) failed. Mostly due to insufficient memory.

  • "Fatal:1012 MALLOC fail (index=%d)\n"

    MALLOC (wrapper of malloc) failed. Mostly due to insufficient memory.

  • "Fatal:1020 array full in %s\n"

    Free element of array does not exist. When you receive this error unexpectedly, check if vce_free_array_object was called.

  • "Fatal:1050 odd buffer size: %d\n"

    Buffer size is not divisible.

  • "Fatal:1080 please set buffer with length of n*256 bytes.\n"

    Buffer length is not a multiple of 256.

  • "Fatal:1090 int-search: MALLOC fail\n"

    Initialization error of init key's search engine. MALLOC (wrapper of malloc) failed. Mostly due to insufficient memory.

  • "Fatal:1091 int-search: init_array fail (index=%d)\n"

    With init key's search engine, initialization failed due to insufficient array.

  • "Fatal:1092 int-search: entry full! index=%d\n"

    With init key's search engine, cannot memorize values any longer.

  • "Fatal:1093 int-search: entry full! index=%d\n"

    With init key's search engine, cannot memorize values any longer.

  • "Fatal:1094 str-search: MALLOC fail\n"

    Initialization error of string key's search engine. MALLOC (wrapper of malloc) failed. Mostly due to insufficient memory.

  • "Fatal:1095 str-search: init_array fail (index=%d)\n"

    With string key's search engine, initialization failed due to insufficient array.

  • "Fatal:1097 str-search entry full! index=%d\n"

    With string key's search engine, cannot memorize values any longer.

  • "Fatal:1098 str-search entry full! index=%d\n"

    With string key's search engine, cannot memorize values any longer.

  • "Fatal:1100 int2-search: MALLOC fail\n"

    Initialization error of 2 integer key's search engine. MALLOC (wrapper of malloc) failed. Mostly due to insufficient memory.

  • "Fatal:1101 int2-search: init_array fail index=%d\n"

    With 2 integer key's search engine, initialization failed due to insufficient array.

  • "Fatal:1102 int2-search: entry full! index=%d\n"

    With 2 integer key's search engine, cannot memorize values any longer.

  • "Fatal:1103 int2-search: entry full! index=%d\n"

    With 2 integer key's search engine, cannot memorize values any longer.

  • "Fatal:1200 tcp: no more connections are availabe\n"

    TCP connection exceeds the maximum number of connection restricted by VCE, so it cannot increase any longer.

  • "Fatal:1201 tcp: bad secret key length: %d\n"

    Length of shared secret-key is abnormal. With Blowfish, it must be either 4 or 56.

  • "Fatal:1202 tcp: bad encryption type specified:%d\n"

    Value of crypt argument of vce_tcpcontext_create must be either 0 or 1.

  • "Fatal:1203 need a parser when creating tcpcontext\n"

    To use vce_tcpcontext_create, at least parser function must be specified. Otherwise it is left out without defining how to process the input from socket.

  • "Fatal:1204 socket() fail addr:%s port:%d os_errno:%s\n"

    socket() system call failed and its status is returned as an error number of C library.

  • "Fatal:1205 gethostbyname() error: %s\n"

    gethostbyname function failed. Also displays error output of C library.

  • "Fatal:1206 gethostbyname() error: %s\n"

    gethostbyname function failed. Also displays error output of C library.

  • "Fatal:1207 bind() error. os_errno:%s addr:%s port:%d\n"

    bind function failed.

  • "Fatal:1208 listen() error. os_errno:%s addr:%s port:%d\n"

    listen function failed.

  • "Fatal:1209 setsockopt() fail. os_errno:%s\n"

    setsockopt function failed.

  • "Fatal:1210 gethostbyname() error: %s\n"

    gethostbyname function failed. Also displays error output of C library.

  • "Fatal:1212 fcntl fail: %s\n"

    fcntl function failed.

  • "Fatal:1213 setsockopt() fail: %s\n"

    setsockopt function failed.

  • "Fatal:1214 keyex_start_client fail on fd:%d\n"

    Cannot start the client side negotiation routine of public key exchange method.

  • "Fatal:1230 cannot write data in %s\n"

    Cannot output to a file.

  • "Fatal:1231 cannot write data in %s\n"

    Cannot output to a file.

  • "Fatal:1250 cannot init vpaddr search engine\n"

    Cannot initialize the search engine called "vpaddr", which is the master database for packet vectorizer.

  • "Fatal:1251 cannot MALLOC master rb.\n"

    Cannot initialize a read buffer for each virtual circuit of packet vectorizer.

  • "Fatal:1252 cannot allocate circuit array\n"

    Cannot initialize an array (VCE array) used to store virtual circuits.

  • "Fatal:1253 cannot initialize int2 search engine:%s\n"

    Cannot initialize a search engine that searches an address and a virtual circuit of packet vectorizer.

  • "Fatal:1256 pointer to tcpcontext is null\n"

    A pointer that redirects tcpcontext is NULL. It has not passed the Sanity check.

  • "Fatal:1260 win32 fatal: gethostbyname() error:%s\n"

    At Winsock, gethostbyname failed.

  • "Fatal:1265 Memory shortage when creating tcpcontext state buffer\n"

    Could not save status management memory for status attached to protocol due to insufficient memory in vce_tcpcontext_create.

  • "Fatal:1268 Cannot reallocate vec's state buffer array. (%s)"

    Packet vectorizer could not reassign status buffer for status attached protocol management (insufficient memory).

  • "Fatal:1270 vce_scandir is not supported on this operating system\n"

    vce_scandir function is not supported with this operating system.

  • "Fatal:1277 setsockopt(SO_REUSEADDR) fail:%s\n"

    Failed to configure SO_REUSEADDR.

  • "Fatal:1280 cannot add ipv4 filter for conn: serious bug? net:%s mask:%s flags:%d\n"

    Could not configure IPv4 filter for conn_t. Possibly because of VCE's internal bug.

  • "Fatal:1281 cannot add ipv4 filter for circ: serious bug? net:%s mask:%s flags:%d\n"

    Could not configure IPv4 filter for circ_t. Possibly because of VCE's internal bug.

Warning Messages

Warning messages indicate that the exhaustion of resource or the deterioration of performance may be experienced in the future even though VCE continues working. It is recommended to avoid warning messages as much as possible by tuning VCE's configuration.

  • "Warning:4000 freeing null pointer in %s\n"

    Attempted to free a null pointer.

  • "Warning:4001 freeing an out-of-bound element in %s address:%p master-address:%p\n"

    An element trying to release is placed outside of memory area of array.

  • "Warning:4002 freeing an oddly-aligned element in %s address:%p master-address:%p sizeof:%u\n"

    An element trying to release is a pointer to buffer which is not an element of array.

  • "Warning:4003 freeing an out-of-bound element in %s address:%p master-address:%p sizeof:%u\n"

    An element trying to release is placed outside of memory area of array.

  • "Warning:4010 server packet has bad length\n"

    Length of packet sent from server is in a strange length.

  • "Warning:4011 server: request queue is full\n"

    Lock request queue of shared memory is full.

  • "Warning:4012 server: being shutting down\n"

    Server of shared memory is currently shut down.

  • "Warning:4013 server: invalid argument\n"

    A parameter which is unknown to server of shared memory was sent from client.

  • "Warning:4014 unknown server error: %d\n"

    A parameter which is unknown to client was sent from server. It has been ignored.

  • "Warning:4015 request timeout for ofs:%u len:%u\n"

    Request from client has timed out.

  • "Warning:4016 unknown reply from server:0x%x\n"

    Can not understand a command from server.

  • "Warning:4017 sender() returns %d not %d\n"

    Could not send records.

  • "Warning:4018 sender() returns %d not %d\n"

    Could not send records.

  • "Warning:4060 notify request timeout to client: ofs:%u len:%u\n"

    Lock request was timed out and a notification was sent to client.

  • "Warning:4061 try_lock error: %s\n"

    Could not acquire a lock for some reason. Other warning messages should be displayed at the same time.

  • "Warning:4062 lock expired ol[%d]\n"

    Lock release command was not received so that the lock itself had timed out.

  • "Warning:4080 bfenc: no space left on %d's ewb\n"

    There was no space in write buffer to write data after encoding so that data could not be sent to socket.

  • "Warning:4100 tcp: only %d connections are usable\n"

    Because it exceeds the maximum number of connection in VCE, only the number of connections displayed can be used.

  • "Warning:4101 tcp: accept() error. os_errno:%s port:%d\n"

    accept() error

  • "Warning:4102 tcp: conn_alloc fail no more accept. max:%d port:%d\n"

    Because it reached the maximum number of connection that VCE can manage, no more TCP connections can be added (or accepted) .

  • "Warning:4103 tcp: acceptwatcher fail fd:%d\n"

    Function specified as acceptwatcher returned an error.

  • "Warning:4104 select() returns %d: %s\n"

    select() returned an invalid value.

  • "Warning:4105 connection timeout. fd:%d sec:%d\n"

    Connection timed out because a client is not working well. The value of time out is specified when vce_tcpcontext_create function is called.

  • "Warning:4200 vecp buffer full. closing client connection.\n"

    Connection to client was terminated since the buffer is full. Generally, when the write buffer for client is full, it is perceived that a client is not working properly and the connection is terminated.

  • "Warning:4201 vecp buffer full. closing client connection.\n"

    Since buffer is full, the connection with client was terminated.

  • "Warning:4202 vecp cannot update search engine on fd:%d\n"

    Cannot register a new client peer address with search engine. This is fatal for client, but it is just a warning for server since a search engine is full.

  • "Warning:4203 vecp buffer full. closing client connection.\n"

    Since buffer is full, the connection with client was terminated.

  • "Warning:4204 vecp: bad protocol magic\n"

    Magic number of protocol is invalid.
    There is a bug with client or there may be a crack.

  • "Warning:4205 vecp: bad version number\n"

    Version number of protocol is invalid.
    There is a bug with client or there may be a crack.

  • "Warning:4206 vecp: bad content length:%d\n"

    Record length of protocol is invalid.
    There is a bug with client or there may be a crack.

  • "Warning:4207 vecsv: bad protocol magic on vp connection\n"

    At VP connection (a connection between a vectorizer and a server), a magic number of protocol is abnormal.

  • "Warning:4208 vecsv: bad version number on vp connection\n"

    At VP connection (a connection between a vectorizer and a server), the version number of protocol do not match.

  • "Warning:4209 vecsv: bad content length:%d\n"

    At VP connection (a connection between a vectorizer and a server), a record length is invalid. Could be a crack.

  • "Warning:4210 vecsv: cannot write data into vp connection\n"

    Cannot transmit since the write buffer for VP connection is full or there is something wrong with connection.

  • "Warning:4211 vecsv: cannot write to circuit buffer.ret:%d start:%d end:%d len:%d\n"

    Cannot write since the write buffer for virtual circuits is full or something is wrong.

  • "Warning:4215 winsock select() returns SOCKET_ERROR, and WSAGetLastError:%d\n"

    With Winsock, select returned an error.

  • "Warning:4214 re-freeing an element in %s address:%p\n"

    Trying to release the same element twice. Possibly the application called free_array_object twice.

  • "Warning:4220 SHA-1 keygen output buffer is smaller than 40byte. It usually needs 40 byte. len:%d\n"

    SHA-1 hash is trying to create a hash for key but the given buffer is less than 40 byte. The buffer must be greater than 40, otherwise outcomes may be different for UNIX and Windows since it would be caught by a bug with Windows _snprintf. 64 is recommended.

  • "Warning:4223 bin16: unparser cannot write full record. try:%d actual:%d\n"

    With a transmission function of bin16 protocol, unable to use a transmission buffer that is sufficient to send a complete record. Therefore, it returned VCE_EWBFULL.

  • "Warning:4225 vce_tcpcontext_create: reached max connection num %d\n"

    It reached the maximum number of connection specified with tcp context.

  • "Warning:4300 vce_conn_io: conn does not have parser function. use vce_tcpcontext_set_conn_parser\n"

    TCP connection "conn" does not have a parser. Configure a parser by using vce_tcpcontext_set_conn_parser to process data input.

  • "Warning:4301 vce_protocol_parser_vecmain_server: circ does not have parser function. use vce_tcpcontext_set_circ_parser\n"

    Virtual circuit "circ" does not have a parser. Configure a parser by using vce_tcpcontext_set_circ_parser to process data input.

  • "Warning:4310 tcpcontext_try_accept: IP source address filter denies this address: %s\n"

    The connection of conn_t was denied by IP source address filter. This is always displayed with verbose mode regardless of FILTER_POLICY_LOG setting. If FILTER_POLICY_LOG is set, it is displayed regardless of verbose mode.

  • "Warning:4311 vectorizer server: IP source address filter denies this address: %s\n"

    The connection of circ_t was denied by IP source address filter. Action is the same with conn_t.

Bug Messages

Most of Bug Messages indicate errors that can not be solved with application. CommunityEngine Inc. will start debugging right away when we receive a bug message report.

  • "Bug:7000 blank-list stack overflow: %s\n"

    Management of a free list of array is abnormal.

  • "Bug:7002 use flag is 0! in %s address:%p\n"

    Sanity check error for the array of vce_show_array_usage function.

  • "Bug:7003 usenum doesn't match in %s stack:%d link_inuse:%d\n"

    The number of count in use is different from the length of stack actually increased.

  • "Bug:7010 tcp: nobuf?\n"

    The free buffer for TCP that should have been free is insufficient.

  • "Bug:7011 tcp: nomem?\n"

    The free memory for TCP that should have been reserved is insufficient.

  • "Bug:7100 vecp: cannot delete search entry\n"

    Not registered with search engine that should have been registered. Therefore it could not be deleted from search engine.

  • "Bug:7101 vecp: cannot find client connection\n"

    Could not find a connection with client in search engine.

  • "Bug:7102 vecp: cannot find client connection\n"

    Could not find a connection with client in search engine.

  • "Bug:7103 vecsv: cannot delete search entry\n"

    Not registered with search engine that should have been registered. Therefore it could not be deleted from search engine.

  • "Bug:7104 vecsv: cannot find client connection\n"

    Could not find a connection with client in search engine.

  • "Bug:7105 vecsv: cannot delete client circuit from search engine.\n"

    Not registered with search engine that should have been registered. Therefore it could not be deleted from search engine.

  • "Bug:7106 vecsv: vce_circ_close fail\n"

    Could not find a virtual circuit that should be closed inside vce_circ_close function. This error could happen when application attempts to close more than twice for the same virtual circuit. Otherwise, it is due to VCE's bug.

  • "Bug:7109 state buffer is full.(accept)\n"

    Status management buffer is full even though connection is not full. Obviously it is VCE's bug.

  • "Bug:7112 connect: statebuf shortage\n"

    Only status management buffer is full when vce_tcpcontext_connect was called. It is VCE's bug.
